我们设计一系列以“build-your-own-x”为主题的实验，引导学生逐步构建一个简易的模型检查工具。这些实验由浅入深、前后衔接，覆盖《Principles of Model Checking》中的核心内容，包括建模并发系统、线性时间属性、分支时间属性、自动机理论、抽象与约简、偏序归约以及概率模型检查。实验使用 Python 实现，强调可操作性和可量化的完成指标。

---

### **实验一：构建 Transition System 与可达状态计算**

- **背景与动机**  
    模型检查的第一步是建立系统的形式模型。Transition System（迁移系统）是描述并发系统行为的基础模型，理解其数据结构与状态空间生成是后续所有验证工作的基石。
    
- **思路**  
    设计一个 `TransitionSystem` 类，用字典或图结构存储状态、迁移关系、初始状态和原子命题标签。实现从文本描述（如邻接表或简单格式）加载模型的功能。编写深度优先/广度优先搜索算法计算所有可达状态，并统计状态数和迁移数。
    
- **关键技术**
    
    - 图的数据结构（邻接表）
        
    - DFS / BFS 算法
        
    - 可达集计算
        
- **完成指标**
    
    1. 能正确解析一个简单的并发系统描述（例如两个进程交替执行的 Peterson 互斥算法的简化模型），并输出所有可达状态列表。
        
    2. 对于给定的两个不同系统，输出的可达状态数目与手工计算结果一致（可使用已知的小例子验证）。
        
    3. 实现状态去重，避免重复状态。

### **实验二：不变性检查（Invariant Checking）**

- **背景与动机**  
    不变性（如“永远互斥”）是最常见的线性时间属性之一，其验证只需要检查每个可达状态是否满足一个命题逻辑公式。该实验将构建基本的属性验证能力。
    
- **思路**  
    实现命题公式的解析器和求值器（支持原子命题、与、或、非）。在实验一的基础上，对每个可达状态计算公式的真值，一旦发现违反公式的状态，输出一条反例路径（从初始状态到该状态的轨迹）。
    
- **关键技术**
    
    - 命题公式解析（递归下降）
        
    - DFS 遍历同时记录路径
        
    - 反例生成
        
- **完成指标**
    
    1. 对 Peterson 互斥算法，检查公式 `¬(crit₁ ∧ crit₂)` 是否成立，程序应报告“通过”并提供验证信息。
        
    2. 人为引入一个违反互斥的状态（如修改迁移），程序应检测到并输出一条从初始状态到坏状态的路径。
        
    3. 可处理多个原子命题，支持公式嵌套。

### **实验三：正则安全属性验证（NFA 与乘积构造）**

- **背景与动机**  
    许多安全属性（如“红色后必须紧跟黄色”）不能简单地用状态公式描述，需要用自动机识别坏前缀。此实验引入有限自动机，通过乘积构造将安全属性验证转化为可达性检查。
    
- **思路**  
    实现 NFA 类（状态、转移、初始/接受状态），支持从简化正则表达式（如 `a* b`）构建 NFA。实现 TS 与 NFA 的同步积（同步乘积），并在积中搜索接受状态是否可达。若可达，输出对应的坏前缀路径。
    
- **关键技术**
    
    - NFA 的数据结构与模拟运行
        
    - 乘积构造
        
    - 积图上的 DFS 搜索
        
- **完成指标**
    
    1. 对交通灯例子（`red` 后必须紧挨 `yellow`），构造对应的 NFA（接受坏前缀），在正常交通灯 TS 上验证属性应通过。
        
    2. 若 TS 被改为“红色后直接变绿”，乘积中应能发现接受状态，并输出反例路径（例如 `red → green`）。
        
    3. 对于给定的正则表达式和 TS，正确报告安全属性是否成立。

### **实验四：LTL 模型检查（自动机法）**

- **背景与动机**  
    LTL 能够表达更丰富的线性时间属性（如“无限经常绿色”）。LTL 模型检查的标准方法是将公式的否定转化为 Buchi 自动机，然后在 TS 与自动机的乘积上检测是否存在无限经常访问接受状态的路径。
    
- **思路**  
    实现 NBA 类（与 NFA 类似，但接受条件为无限经常）。提供一个有限但实用的 LTL 到 NBA 转换（可手动编码常用模式或使用简化的 Tableau 构造）。实现 TS 与 NBA 的乘积，并利用嵌套 DFS 检测是否存在包含接受状态的循环。若存在，输出反例路径（包含循环）。
    
- **关键技术**
    
    - NBA 的数据结构
        
    - 嵌套深度优先搜索（nested DFS）
        
    - 接受循环检测
        
- **完成指标**
    
    1. 对 TS 模拟红绿灯（只有红绿两色），检查 LTL 公式 `□♦green`（无限经常绿灯）。应报告通过（若 TS 确实无限经常变绿）或失败（若 TS 可能永远变绿一次后不再变）。
        
    2. 对通信协议例子，检查 `□(send → ♦ack)`。应能给出正确结果。
        
    3. 对于违反属性的情况，输出的反例路径应包含循环部分（例如重复发送但不收到确认的循环）。

### **实验五：CTL 模型检查**

- **背景与动机**  
    与 LTL 不同，CTL 支持分支时间属性，例如“所有计算都无限经常可能回到初始状态”。CTL 模型检查基于固定点计算，是理解抽象解释和模型检查算法的关键。
    
- **思路**  
    实现 CTL 公式的递归满意度计算。支持基本算子 EX、EU、EG，并利用 De Morgan 律推导出其他算子（如 EF、AG）。需要实现状态集合的并、交、补以及前置集合计算（`Pre(C)`）。对于 EU 和 EG 使用最小/最大固定点迭代。
    
- **关键技术**
    
    - 状态集合的位集表示
        
    - 前置函数 `Pre`
        
    - 固定点迭代算法
        
- **完成指标**
    
    1. 对 Peterson 算法，检查 CTL 公式 `∀□(¬crit₁ ∨ ¬crit₂)`（互斥），应报告所有初始状态满足。
        
    2. 检查公式 `∀♦(wait₁ → ∀♦crit₁)`（进程1不会饥饿），若不满足应提供反例（一条路径）。
        
    3. 能正确生成 EU 的满足集，并与手工计算结果对比（可用小型例子验证）。

### **实验六：抽象技术——Bisimulation 最小化**

- **背景与动机**  
    状态空间爆炸是模型检查的主要障碍。Bisimulation 等价可以合并行为相同的状态，大幅缩减模型尺寸，且保持 CTL* 性质。此实验实现经典的分区细化算法，构造最小化迁移系统。
    
- **思路**  
    输入 TS，首先按原子命题划分初始块。然后反复根据迁移关系细化：若块 B 中的状态在迁移到块 C 的能力上不一致，则拆分 B。使用队列和栈优化算法（如 Paige-Tarjan 算法）。得到最小化 TS 后，可在其上重新验证属性，并与原 TS 的结果对比。
    
- **关键技术**
    
    - 分区细化（partition refinement）
        
    - 基于分裂子的高效实现（使用计数器）
        
    - 最小化 TS 的构造
        
- **完成指标**
    
    1. 对给定的 TS（例如两个互斥进程的 TS），计算其 bisimulation 等价类，最小化后状态数应明显减少（例如从 8 个状态减少到 4 个）。
        
    2. 验证原 TS 与最小化 TS 在若干 CTL 公式上的结果一致。
        
    3. 实现与实验五的集成：允许用户选择是否先最小化再验证。

### **实验七：偏序归约（Partial Order Reduction）**

- **背景与动机**  
    并发系统的状态空间常因进程交错而爆炸。偏序归约利用动作的独立性，只探索部分交错，同时保证 LTL\X 属性。此实验实现“ample set”方法，在状态空间生成中动态剪枝。
    
- **思路**  
    在实验一的 TS 生成过程中，对每个状态计算 ample 集（基于进程的局部条件和依赖分析）。仅探索 ample 集中的动作。需要实现依赖关系分析（根据共享变量、通道等）并满足 ample 集的四个条件（非空、依赖、无分歧、循环条件）。可通过强化的循环条件（A4'）简化实现。生成简化 TS 后，验证其与原 TS 在 LTL\X 公式上的一致性。
    
- **关键技术**
    
    - 动作独立性分析
        
    - ample 集计算（基于静态信息）
        
    - DFS 中动态调整 ample 集（循环条件）
        
- **完成指标**
    
    1. 对两个独立递增计数器的并发程序，使用偏序归约后生成的状态数应远小于完全探索（例如从 O(n²) 降至 O(n)）。
        
    2. 检查 LTL\X 公式（如 `□(count₁ < 10)`）在简化 TS 和原 TS 上结果一致。
        
    3. 提供开关选项：用户可选择是否启用偏序归约。

### **实验八（可选/拓展）：概率模型检查（离散时间马尔可夫链与 PCTL）**

- **背景与动机**  
    许多系统包含随机行为（如丢包、随机算法），需要定量分析概率性质。此实验将模型从非确定迁移系统扩展到离散时间马尔可夫链（DTMC），并实现 PCTL 模型检查的基础算法。
    
- **思路**  
    实现 DTMC 数据结构（状态、转移概率矩阵、初始分布）。实现 PCTL 公式的递归求解，包括：
    
    - 概率算子 `P~p(ϕ)`：计算路径公式 ϕ 的概率（如 `P>0.5(♦target)`），并比较阈值。
        
    - 路径公式 ⃝Φ、Φ U Ψ 的概率计算（解线性方程组或迭代法）。
        
    - 对于 `Φ U Ψ`，实现求解线性方程组（高斯消元或 Jacobi 迭代）。  
        提供图形化或文本形式输出每个状态满足的概率。
        
- **关键技术**
    
    - 稀疏矩阵存储
        
    - 线性方程组求解（迭代法）
        
    - PCTL 语义解释
        
- **完成指标**
    
    1. 对简单的通信协议 DTMC（如丢包重传模型），计算 `P=1(♦delivered)`（最终送达的概率应为 1）。
        
    2. 对 Knuth 的掷骰子模拟 DTMC，计算 `P=1/6(♦i)` 对 i=1..6 均成立。
        
    3. 能处理带有界 until 的公式（如 `P<0.01(♦^5error)`），并与理论值比较（误差小于 1e-3）。

### **实验整合建议**

每个实验独立但依赖前置实验的基础代码。建议学生以面向对象方式组织代码，将 TS、NFA、NBA、DTMC 等模型作为独立模块，模型检查算法作为策略类。实验六至八可基于前序实验的 TS 或 DTMC 实现。完成所有实验后，学生将拥有一个支持 LTL、CTL、偏序归约、最小化和概率分析的简易模型检查工具，并能验证典型并发系统的常见属性。
